# AI Hedge Fund

An educational, multi-agent trading system: thirteen LLM "persona"
investors (Buffett, Munger, Cathie Wood, Duan Yongping, Druckenmiller, …) plus
a full quantitative stack (signals, portfolio optimizers, validation,
event-study) decide whether to BUY / SELL / HOLD / SHORT each ticker.

Markets supported: **US, Hong Kong, China A-share** (Shanghai +
Shenzhen). UI / output supports both **English and Simplified Chinese**.

## Install

This project uses [**uv**](https://docs.astral.sh/uv/) for dependency
and project management.

```bash
# 1. Clone
git clone https://github.com/loadchange/ai-hedge-fund.git
cd ai-hedge-fund

# 2. Install uv (if you don't have it)
curl -LsSf https://astral.sh/uv/install.sh | sh

# 3. Install Python deps + project (editable)
uv sync

# 4. Configure API keys
cp .env.example .env
# Edit .env — at minimum set ONE LLM key (OPENAI_API_KEY, ANTHROPIC_API_KEY,
# DEEPSEEK_API_KEY, MOONSHOT_API_KEY, GROQ_API_KEY, …) plus optionally
# FINANCIAL_DATASETS_API_KEY for richer US data.
```

The `.env.example` lists every key the system understands. You only need
the ones for the providers you actually want to use. CN/HK markets work
without any data-API key (they use free Tencent + akshare endpoints).

## Run

### Hedge fund (`src/main.py`)

Single-day, multi-agent analysis on one or more tickers.

```bash
# Default: all analysts, today's date
uv run python src/main.py --tickers AAPL,MSFT,NVDA --model mimo-v2.5-pro

# A subset of analysts, a fixed date window
uv run python src/main.py --tickers AAPL --model mimo-v2.5 \
  --analysts warren_buffett,duan_yongping,charlie_munger \
  --start-date 2025-01-01 --end-date 2025-04-01

# Chinese output
uv run python src/main.py --tickers 600519.SS --lang zhCN \
  --analysts duan_yongping,warren_buffett --model mimo-v2.5-pro

# Local LLM via Ollama
uv run python src/main.py --tickers AAPL --ollama
```

Output: per-ticker agent table (signal + confidence + reasoning),
trading decision, and portfolio summary.

### Backtester (`src/backtester.py`)

Re-runs the same multi-agent workflow on every business day in the
window, then reports performance with **transaction costs**.

```bash
# Default: 10 bps fixed cost, all analysts
uv run python src/backtester.py --tickers AAPL,MSFT \
  --model mimo-v2.5-pro \
  --start-date 2025-01-01 --end-date 2025-02-01 \
  --analysts warren_buffett,duan_yongping

# Spread + sqrt-impact cost model
uv run python src/backtester.py --tickers AAPL --cost-model spread --cost-bps 5

# Frictionless sanity check
uv run python src/backtester.py --tickers AAPL --cost-bps 0
```

The summary at the end prints **Total Transaction Costs** broken down
by buy / sell / short / cover, plus Sharpe / Sortino / Max DD.

> ⚠️ Backtester cost scales with `analysts × tickers × days × LLM
> price`. For experimentation prefer 1 ticker × 2-3 analysts × 1-2
> weeks; full sweeps belong in CI runs with budget allocated.

### Validation CLI (no LLM cost)

Quantitative signal evaluation with **Combinatorial Purged
Cross-Validation** (CPCV), **Probability of Backtest Overfitting**
(PBO), and **Deflated Sharpe Ratio**. Pure pandas/numpy — no LLM
calls, runs in seconds.

```bash
# One signal × one ticker
uv run python -m src.validation.cli evaluate \
  --signal momentum --ticker AAPL \
  --start 2023-01-01 --end 2025-04-01 \
  --n-splits 8 --n-test-splits 2 --rolling-window 180

# Sweep multiple signals across multiple tickers
uv run python -m src.validation.cli evaluate \
  --signal trend,mean_reversion,momentum,volatility,stat_arb \
  --ticker AAPL,MSFT,NVDA \
  --start 2023-01-01 --end 2025-04-01 \
  --rolling-window 180
```

Each row prints `IS Sharpe / OOS Sharpe / PBO / DSR`. Full per-split
statistics land in `validation_results.json`.

### LLM cost cheatsheet

| Command | Hits the LLM? | Notes |
|---|---|---|
| `src/main.py` | **Yes** | One LLM call per persona-agent per ticker, plus the portfolio manager. 5 tickers × all analysts ≈ 100 calls. |
| `src/backtester.py` | **Yes (per day)** | Same workflow per business day → cost scales as `analysts × tickers × days`. |
| `python -m src.validation.cli evaluate` | **No** | Pure quant: `BaseSignal.compute_from_prices()` on rolling windows + CPCV/PBO. |
| `from src.{signals,risk,portfolio,validation,event_study,features} import *` | **No** | All quant modules are LLM-free Python you can call from notebooks. |

## Issue bot

GitHub Action ([`.github/workflows/issue-bot.yml`](.github/workflows/issue-bot.yml))
turns issues into runnable hedge-fund / backtest jobs.

**Trigger an analysis from a GitHub issue:**

1. Open an issue using the **🤖 Ticker analysis** or **📈 Backtester**
   template (4 templates total: 2 modes × `lang-zh` / `lang-en`).
2. The template auto-applies a `bot-ticker` or `bot-backtester` label
   plus a language label.
3. The Action calls the configured AI to parse arguments out of the
   issue title + body, then runs `uv run python src/main.py` or
   `src/backtester.py`.
4. Result is posted as a Markdown comment (proper tables, not Rich
   ANSI), the issue is closed automatically, and full structured
   output (`comment.md`, `result.json` / `backtest_metrics.json`,
   `portfolio_values.json`) is uploaded as a workflow artifact.

**Required repo secrets**: `AI_BASE_URL`, `AI_API_KEY`, `AI_MODEL` (the
last must exist in [`src/llm/api_models.json`](src/llm/api_models.json)).
Optional: `FINANCIAL_DATASETS_API_KEY`.

The parser also enforces a **size cap** (≈ 200 estimated LLM calls)
that refuses requests with no realistic chance of finishing inside the
30-minute workflow timeout — and the failure path correctly replies on
both `failure()` and `cancelled()` so timeouts never silently die.

## Quantitative modules

Six standalone Python packages complement the LLM agents with classical
quant tooling. Each is importable directly (no LangGraph required), so
you can use them from notebooks, custom scripts, or your own pipelines.

| Module | Purpose | Key entrypoints |
|---|---|---|
| [`src/signals/`](src/signals/) | `BaseSignal` ABC + 8 concrete signals (5 technical: trend / mean_reversion / momentum / volatility / stat_arb; 3 fundamental: value / quality / earnings_surprise). Each signal returns a `SignalResult` with a signed score in `[-1, +1]`. | `from src.signals import SIGNAL_REGISTRY, MomentumSignal` |
| [`src/risk/`](src/risk/) | Volatility / correlation matrices, drawdown analytics, scenario stress (2008 / 2020 / 2022 / 2025), Kelly + vol-targeted sizing. | `from src.risk import drawdown_stats, default_scenarios` |
| [`src/portfolio/`](src/portfolio/) | Three cvxpy optimizers (mean-variance / risk parity / Black-Litterman), Ledoit-Wolf shrinkage, Marchenko-Pastur eigenvalue cleaning, and `build_views_from_signals` that turns analyst signals into BL views. | `from src.portfolio import MeanVarianceOptimizer, BlackLittermanOptimizer` |
| [`src/validation/`](src/validation/) | CPCV + PBO + Deflated Sharpe + a CLI runner (see above). | `python -m src.validation.cli evaluate …` |
| [`src/event_study/`](src/event_study/) | Market model α/β fit, abnormal returns, CAR / CAAR, t-test + Wilcoxon. | `from src.event_study import fit_market_model, compute_car` |
| [`src/features/`](src/features/) | SUE, PEAD drift, KPI YoY momentum (z-scored), pairwise lead-lag, Granger causality. | `from src.features import standardised_unexpected_earnings, granger_causality` |

The persona agents and quant modules **work side-by-side**: a hedge-fund
run still produces the colourful agent-by-agent rationale, and the
same `analyst_signals` payload flows straight into
`build_views_from_signals` to drive the Black-Litterman optimizer or
into `src.validation` to ask "is this signal actually OOS-robust?"

## Markets & data sources

The hedge fund routes data requests to multiple providers based on the
**ticker format**:

| Market | Ticker examples | Detection rule |
|---|---|---|
| US | `AAPL`, `MSFT`, `NVDA` | default |
| Hong Kong | `9988.HK`, `0700.HK` | `.HK` suffix |
| China A-share (Shanghai) | `600519.SS`, `600519` | `.SS` suffix or 6-digit numeric starting with `6` |
| China A-share (Shenzhen) | `002594.SZ`, `000001` | `.SZ` suffix or 6-digit numeric starting with `0` / `3` |

Detection lives in [`src/data/sources/base.py`](src/data/sources/base.py)
(`classify_ticker`).

`DataSourceManager` ([`src/data/sources/manager.py`](src/data/sources/manager.py))
orchestrates these providers with **priority routing**, **fallback**,
**5-minute rate-limit cooldowns**, and **price cross-validation**
(warns when two sources differ by > 2%):

| Source | Coverage | Notes |
|---|---|---|
| `financialdatasets` | US prices + financials + line items | Requires `FINANCIAL_DATASETS_API_KEY`; richest data |
| `yfinance` | US / HK prices + financial metrics | Free; rate-limited |
| `akshare` | US / HK / CN prices + financials | Free; uses Sina / Eastmoney upstream |
| `tencent` | US / HK / CN realtime quotes (market-cap, PE, PB) | Free; HK / CN financial metrics are enriched from this |

For A-share line items there's also a **fallback** in
`src/tools/api.py` that derives missing fields
(`shareholders_equity`, `total_liabilities`, `revenue`,
`outstanding_shares`) from akshare's financial-indicator table when
financialdatasets doesn't cover the ticker.

## Internationalisation

Add `--lang zhCN` to any CLI to switch the entire output (progress
bars, agent names, signals, table headers, reasoning) to Simplified
Chinese. Supported: `en` (default) and `zhCN`.

Implementation in [`src/i18n.py`](src/i18n.py):

- Status messages, table headers, action / signal enums, and the 14
  investor-agent display names are translated via lookup dictionaries.
- The chosen language is injected into every LLM system prompt
  (`get_lang_instruction`) so model-generated reasoning matches.
- A post-call sanitiser (`_sanitize_reasoning_fields` in
  `src/utils/llm.py`) walks the Pydantic response and converts any
  JSON-shaped reasoning into natural-language text via
  `summarize_json_reasoning`, so the output table never shows raw JSON.

## Network & proxy notes

- **Proxy bypass**: domestic data endpoints (Sina, Tencent at
  `web.ifzq.gtimg.cn`) often break behind a proxy. The akshare adapter
  temporarily clears `HTTP_PROXY` / `HTTPS_PROXY` for these hosts via
  `_no_proxy_for(PROXY_BYPASS_DOMAINS)` in
  [`src/data/sources/akshare_src.py`](src/data/sources/akshare_src.py).
  Add hosts to `PROXY_BYPASS_DOMAINS` if you encounter additional
  ones.
- **SSL retry**: when the financialdatasets request fails with an SSL
  error through a proxy, `tools/api.py:_make_api_request` retries once
  without the proxy.
- **Concurrency lock**: akshare ships with `py-mini-racer` (a V8
  isolate) for some Sina endpoints whose pool is **not** thread-safe;
  a module-level lock in `akshare_src.py` serialises every `ak.*`
  call so concurrent agent fan-out doesn't crash.
- **Anthropic via gateway**: set `ANTHROPIC_BASE_URL` in `.env` to
  route Claude calls through a proxy. Extended thinking is disabled
  by default in `src/llm/models.py` so JSON-mode parsing stays
  reliable.
- **OpenAI-compatible gateways** (e.g. xiaomi MiMo, OpenRouter): set
  `OPENAI_API_BASE` alongside `OPENAI_API_KEY`. Note that all
  `provider: "OpenAI"` entries in `src/llm/api_models.json` share the
  same env vars, so picking a model-only gateway will affect the GPT
  entries too.
